Demo 3 — Always Has a Current Value (.getValue())

Unlike a plain Subject, a BehaviorSubject always holds the last value in memory. You can read it synchronously at any time — no subscription needed. This makes it ideal for state management (e.g. current user, theme, shopping cart).
