How Lazy Loading Works

Instead of importing the component directly (eager load), the route uses loadComponent with a dynamic import(): 

{
  path: 'enroll',
  loadComponent: () =>
    import('./enroll/enroll')
      .then(m => m.Enroll)
}

Eager vs Lazy

  • Eager (normal import) – Component is bundled into the main chunk. Downloaded on first page load regardless of whether the user visits that route.
  • Lazy (loadComponent) – Component is split into a separate chunk. Only downloaded when the user navigates to that route. Improves initial load time.

Verify it in DevTools

Open the browser's Network tab → filter by JS → click the Enroll tab for the first time. You will see a new .js chunk appear in the network requests.
