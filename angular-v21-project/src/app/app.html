🚀 Angular 21 - Session 4

In-depth exploration of services, dependency injection, RxJS, and asynchronous programming in Angular

Sibling Communication via Shared Service (MessageService)

Sender and Receiver are siblings sharing state through MessageService. Three approaches are demonstrated side-by-side: plain property (non-reactive snapshot) · BehaviorSubject subscription (reactive, manual unsubscribe) · async pipe (reactive, auto-unsubscribe).

Signal-based Parent ↔ Child Communication

Demonstrates modern Angular signal APIs: input() replaces @Input() · output() replaces @Output() EventEmitter · Parent holds state in signal() and passes the unwrapped value down.

@ViewChild — Imperative Child Access

@ViewChild gives the parent a direct TypeScript reference to a child component instance. The reference is resolved after the view renders and is available in ngAfterViewInit(). Use it to call child methods or read child properties imperatively.

Subject — Fire-and-Forget Observable Stream

Subject emits only to current subscribers — no initial value, no replay. Unlike BehaviorSubject, a late subscriber misses all previous emissions. Each message is pushed into an array so the UI accumulates a full history. Subscription is created in ngOnInit and torn down in ngOnDestroy.

BehaviorSubject — Stateful Stream with Value History

BehaviorSubject emits its current held value immediately on subscribe, then every subsequent next() emission. Demonstrates the full Observer object pattern: { next, error, complete } vs shorthand callback. "Get Current Value" reads synchronously via getValue() without subscribing.

HTTP CRUD — HttpClient with Signals

GET · POST · PUT · DELETE using HttpClient and the JSONPlaceholder API. All state managed with signal(). Service methods return Observable<T> — component subscribes with the full { next, error } Observer pattern.
