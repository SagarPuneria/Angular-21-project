5.1 Core Routing — SPA Navigation RouterOutlet · RouterLink · RouterLinkActive · redirect / wildcard / child routes

Angular Router maps URLs to components without full page reloads. <router-outlet> is the placeholder where matched components render. routerLink replaces href for client-side navigation. routerLinkActive adds a CSS class when the route is active. Route types: redirect (pathMatch: 'full'), specific, wildcard (**), and child routes. The Courses route uses child routes — its own nested <router-outlet> renders Overview and Details.

▶ Try it: 🏠 Home 👥 About 📚 Courses (child routes)
5.2 Lazy Loading — loadComponent() Dynamic import() · separate JS chunks fetched on first navigation · smaller initial bundle

Lazy loading defers the download of a component's JS chunk until the user actually navigates there. loadComponent: () => import('./path').then(m => m.MyComponent) is a dynamic import — the module is fetched on demand, not bundled at startup. Enroll is the classic example: only 1 in 10 users clicks it, so it shouldn't be in the initial bundle. In the browser Network tab you'll see a new chunk appear when you first visit a lazy route.

▶ Try it: 🎓 Enroll lazy 🛒 Products lazy
5.3 Route Guards — canActivate Functional CanActivateFn · block unauthenticated access · returnUrl redirect

A canActivate guard runs before a route is entered. The functional CanActivateFn (Angular 17+) uses inject() — no class needed. Dashboard is protected: unauthenticated users are redirected to /login?returnUrl=/dashboard. After login, the Login component reads returnUrl from query params and navigates there automatically. Guard returns true to allow, false to block, or a UrlTree to redirect.

▶ Try it: 🔐 Dashboard (guarded) 🔑 Login (admin / 1234)
5.4 Route Guards — canDeactivate Functional CanDeactivateFn · unsaved-changes protection · form.dirty check

canDeactivate runs before the user navigates away from a route. The guard calls component.hasUnsavedChanges() — if the method returns true, the browser shows a confirmation dialog. This prevents accidental loss of form data. The component implements a HasUnsavedChanges interface; the guard is a plain CanDeactivateFn<HasUnsavedChanges> function — no class, no decorator needed.

▶ Try it: 🚧 canDeactivate Demo — type, then navigate away
5.5 Route Parameters & withComponentInputBinding() :id segment auto-bound to input() signal · computed() lookup · no ActivatedRoute

Route params embed data directly in the URL (/products/2). With withComponentInputBinding() enabled in app.config.ts, the :id segment is automatically bound to a matching input<string>('id') on the component — no ActivatedRoute subscription needed. A computed() signal then looks up the product in the static array and re-runs whenever the param signal changes.

▶ Try it: 🛒 Product List → click a product for detail
5.6 Reactive Forms — FormBuilder, FormGroup, Validators Typed controls · state flags (.dirty / .touched) · patchValue / setValue · service-backed flow

Reactive Forms define the form model in TypeScript. inject(FormBuilder) creates FormGroup with typed FormControls and Validators. Errors show only after the user has touched a field (not on load). patchValue() updates specific fields; setValue() requires all fields. getRawValue() returns a typed object including disabled controls. The Register → Summary flow demonstrates sharing data across routes via a singleton RegistrationDataService (no HTTP needed).

▶ Try it: 🧾 Registration Form 📄 Summary (post-submit)
5.7 FormArray — Dynamic Repeated Fields Variable-length field groups · push() / removeAt() · formArrayName + [formGroupName]

FormArray manages a variable number of identical field groups — e.g. multiple phone numbers or skills. Use formArrayName on the container, then [formGroupName]="i" on each child (by $index). push() adds a new group; removeAt(index) removes one. A typed getter avoids casts in the template: get phones() { return this.form.get('phones') as FormArray; }

▶ Try it: 🧩 FormArray Demo (phones + skills)
5.8 FormControl Advanced & Signal Forms (Angular 21) nonNullable · toSignal vs computed() · experimental signal-based forms without FormControl

{ nonNullable: true } on a FormControl changes reset() to restore the initial value instead of setting null. Never use computed() to read form.valid / form.dirty — those are plain class getters, not signals, so computed() won't re-run. Use toSignal(form.events.pipe(map(...))) instead. Signal Forms (Angular 21, experimental) replace FormControl entirely with plain signal() state and computed() for derived validity — no ReactiveFormsModule needed.

▶ Try it: 🎛️ Controls Demo (nonNullable + toSignal) ⚡ Signal Form (experimental)
